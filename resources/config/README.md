# 配置文件

在 Linux 系统中，能经常看到一些配置目录，里面的配置文件名字如：`10-desplay.conf` `20-keyboard.conf` `30-bluetooch.conf`...
这种配置。

这种程序并不认定某个文件作为配置文件，而是认定某个目录，目录下的所有 `*.conf` 文件都作为配置文件，配置文件名叫什么不重要，但是名字顺序决定了程序加载配置文件的顺序。

排在前面的文件会先解析，文件的配置项会进行合并，如果配置项 `KEY` 一样，则后面的覆盖掉前面的配置项。

其实文件名以 `10` `20` `30`... 这些数字开头，既给用户一种直观的感觉，又符合字典序的顺序。数字不连续是为了留出空位，便于以后往某个区间插入配置文件。

例如下列目录的配置文件都用到了此风格：

- 系统内核启动参数目录：`/etc/sysctl.d/`

- 系统文件描述符配置目录：`/etc/security/limits.d/`

- 字体配置目录：`/etc/fonts/conf.d/`

- Ubuntu 网络配置目录：`/etc/netplan/`
